# 🗺️ Koharu-TH — แผนงานสถาปัตยกรรมและการพัฒนาผลิตภัณฑ์ (ROADMAP & ARCHITECTURE)

ยินดีต้อนรับสู่คู่มือวางแผนงานพัฒนาและสถาปัตยกรรมระบบของ **Koharu-TH** เอกสารฉบับนี้จัดทำขึ้นเพื่อรวบรวมแผนงานอ้างอิงและสรุปสถานะการพัฒนาเปรียบเทียบกับโปรเจกต์ต้นทาง (Upstream) เพื่อความโปร่งใสและเป็นระเบียบเรียบร้อยของโครงการ

---

## 📌 PART 1: สรุปการควบรวมแผนเอกสารอ้างอิงทั้งหมด (Docs & Roadmap Consolidating)

ส่วนนี้เป็นการรวบรวมแผนงานเชิงลึกเพื่อพัฒนาเครื่องยนต์ประมวลผล AI ของ Koharu-TH ในเวอร์ชันถัดๆ ไป โดยแยกออกจากหน้าคู่มือการติดตั้งทั่วไปเพื่อรักษาความคลีนของไฟล์หลัก

### 1.1 ปัญหาการแยกประเภทไบนารีตามสถาปัตยกรรม GPU (GPU Architecture Blocker)
* **สถานะปัจจุบัน:** **BLOCKED (อยู่ระหว่างรอต้นทาง)**
* **รายละเอียดปัญหา:** ปัจจุบันระบบจำเป็นต้องคอมไพล์และบิลด์ไฟล์สำหรับประมวลผลการ์ดจอแยกทีละรุ่นสถาปัตยกรรม (Turing SM 7.5, Ampere SM 8.0, Ada SM 8.9, Blackwell SM 12.0) เพื่อดึงประสิทธิภาพจากหน่วยประมวลผลการ์ดจอให้ได้สูงสุด ส่งผลให้ตัวติดตั้งมีขนาดใหญ่และแยกย่อยตามกลุ่มผู้ใช้งาน
* **แนวทางแก้ไข:** ทางเรากำลังติดตามและรอให้โปรเจกต์ต้นทาง (`mayocream/koharu`) ปรับปรุงสถาปัตยกรรมการจัดการโมเดลหลังบ้าน (มีหมุดหมายในโครงสร้างหลัก `v1.2.0`) ให้เรียบร้อย เพื่อที่เราจะสามารถนำการปรับปรุงดังกล่าวมาออกแบบระบบตัวติดตั้งเดี่ยว (Single-Binary) ผ่านระบบ PTX JIT หรือ Vulkan ได้ในฝั่งของเราอย่างรวดเร็ว

---

### 1.2 แผนพัฒนาระบบลบล้างข้อความ 3 ระดับ (AI Inpainting 3-Tier Roadmap)
เพื่อรองรับทรัพยากรและเครื่องคอมพิวเตอร์ของผู้แปลทุกกลุ่ม ตั้งแต่โน้ตบุ๊กทำงานทั่วไปจนถึงเวิร์กสเตชันการ์ดจอสูง เราได้วางโครงสร้างโมเดลลบอักษรคลีนภาพ (Inpainting) ไว้ 3 ระดับ:

| ระดับ (Tier) | เครื่องยนต์และโมเดลหลัก | ความต้องการระบบ | คุณภาพและจุดเด่น | สถานะการพัฒนา |
| :--- | :--- | :--- | :--- | :--- |
| **Tier 1: Basic** | **LaMa Local** (Resolution-Robust Large Mask Inpainting) | ต่ำมาก (CPU ทั่วไป หรือการ์ดจอประหยัด) | รวดเร็ว (1-3 วินาที) เหมาะสมกับกล่องคำพูดสีพื้นหรือพื้นหลังเรียบง่าย ทำงานออฟไลน์ 100% | 🟢 **เสร็จสมบูรณ์/ใช้งานปัจจุบัน** |
| **Tier 2: Medium** | **Stable Diffusion Inpainting** (Quantized / OpenVINO Optimized) | ปานกลาง (การ์ดจอ VRAM 6-8GB หรือ CPU ผ่านการบีบอัด) | สามารถสร้างฉากหลัง ลบลายเส้น และสร้างฉากเสริมบนพื้นที่ซับซ้อนปานกลางได้โดยตรงในเครื่อง | 🟡 **แผนพัฒนาในรุ่นถัดไป** |
| **Tier 3: Extreme** | **FLUX.1 / Klein Generative Fill** (โหมดคลาวด์ Cloud API หรือ VRAM 12GB+) | สูงมากในการรันแบบ Local หรือใช้อินเทอร์เน็ตในการส่งคำสั่งคลาวด์ | **คุณภาพสูงเนียนกริบเทียบเท่า Generative Fill ของ Photoshop** ทำงานได้ใน 1 วินาทีผ่านคลาวด์ (Fal.ai / Replicate API) เหมาะกับพื้นหลังยากๆ | 🟡 **แผนพัฒนาในรุ่นถัดไป** |

### 1.3 แผนพัฒนาระบบตรวจจับข้อความอัจฉริยะแบบปรับตัว (Adaptive Text Detection Roadmap)
เราวางเป้าหมายอัปเกรดระบบตรวจจับข้อความ (OCR & Text Detection Pipeline) ให้ประมวลผลได้อย่างชาญฉลาดและรวดเร็วขึ้น:
* **การคำนวณคู่ขนานและส่งต่อลำดับขั้น (Cascade & Parallel Detection):** ปรับจูนระบบให้ส่งหน้าที่การตรวจจับพื้นฐาน (เช่น กล่องคำพูดธรรมดา) ไปยังโมเดลประมวลผลเบาอย่าง `comic_text_detector` เพื่อความเร็วสูงสุด และส่งงานยาก (เช่น ตัวอักษรเขียนมือเอฟเฟกต์มังงะ หรือตัวอักษรศิลป์ที่คาบเกี่ยวพื้นหลัง) ไปรันคู่ขนานกับโมเดลใหญ่อย่าง `Anime Text YOLO` เฉพาะจุดโดยอัตโนมัติ
* **ระบบเลือกขนาดโมเดลอัตโนมัติ (Auto-Scaling Variant Heuristics):** 
  * วิเคราะห์จาก **ความละเอียดรูปภาพ:** หากภาพมีความละเอียดต่ำ/ปานกลางจะรันโมเดลขนาดเล็ก (`N` / `S`) เพื่อประหยัดพลังงาน หากรูปภาพเป็นระดับ 4K สแกนคมชัดสูงจะสเกลขึ้นไปใช้โมเดลขนาดใหญ่ (`X`) เพื่อความแม่นยำสูงสุด
  * วิเคราะห์จาก **ฮาร์ดแวร์ของผู้ใช้งาน:** ปรับขนาดโมเดลเป็นระดับประหยัดหากรันบน CPU หรือการ์ดจอออนบอร์ด เพื่อป้องกันการค้าง และสเกลเป็นระดับความแม่นยำสูงเมื่อพบการ์ดจอแยกที่มีสเปกประมวลผลเพียงพอ

## 📌 PART 2: ระบบที่ฝั่งโค้ดแม่ "ยังไม่ได้เริ่มต้นเลยแม้แต่น้อย" (Upstream 0% Progress Systems)

จากการตรวจสอบซอร์สโค้ดและรายงานการตรวจสอบระบบทุกกิ่งของฝั่งต้นทาง (Upstream/EarthWL) พบว่ามีนวัตกรรมและระบบการทำงานของ **Koharu-TH** หลายระบบที่ **ฝั่งโค้ดแม่ยังไม่เคยได้เริ่มต้นพัฒนาหรือวางฐานระบบใดๆ เลยแม้แต่น้อย (0% Completion)** โดยระบบเหล่านี้ถูกขับเคลื่อนและใช้งานได้จริงบนเวอร์ชันนี้ของ Koharu-TH แล้ว:

### 2.1 ระบบร้านค้าเสริมในโปรแกรมแบบไม่ต้องติดตั้งแยก (Premium In-App Addon & Language Store)
* **สถานะของต้นทาง:** ❌ **ยังไม่ได้เริ่มต้น (0%)**
* **ฟีเจอร์ใน Koharu-TH:** เราได้วางระบบสโตร์ส่วนเสริมแบบครบวงจร มีหน้าอินเตอร์เฟสสวยงามให้กดดาวน์โหลดและเปิดใช้งานโมเดลภาษา, ธีมแอปพลิเคชันพรีเมียม และแพ็คเกจฟอนต์ได้โดยตรงผ่านแอปพลิเคชัน ไม่จำเป็นต้องนำไฟล์ไปวางแยกในโฟลเดอร์ให้เกิดความยุ่งยากอีกต่อไป

### 2.2 ระบบสลับและแจ้งเตือนภาษาแบบทันใจ (Relaunch Alert Alert switch & Switcher)
* **สถานะของต้นทาง:** ❌ **ยังไม่ได้เริ่มต้น (0%)**
* **ฟีเจอร์ใน Koharu-TH:** มีการพัฒนาระบบสลับข้ามภาษาและตรวจสอบการเปลี่ยนค่าภาษาของแอปพลิเคชัน เมื่อมีการเปลี่ยนแปลง แอปจะทำการรีสตาร์ทตัวประมวลผลเบื้องหลังให้อัตโนมัติเพื่อแสดงภาษาใหม่ได้ทันทีโดยไม่ต้องปิดโปรแกรมแบบแมนนวล

### 2.3 ปุ่มปรับแต่งความหนา-ความเอียงฟอนต์ระดับพรีเมียม (Photoshop-style Text Bold/Italic Native Controls)
* **สถานะของต้นทาง:** ❌ **ยังไม่ได้เริ่มต้น (0%)**
* **ฟีเจอร์ in Koharu-TH:** มอบแผงควบคุมสไตล์โปรแกรม Photoshop (A/A, VA controls) ให้ปรับค่าความหนา (Bold), ความเอียง (Italic), ระยะห่างระหว่างอักษร และขนาดตัวหนังสือแบบไดนามิก ปรากฏการเปลี่ยนแปลงสดบนแคนวาสขณะปรับ

### 2.4 ระบบอัปเดตแอปพลิเคชันอัตโนมัติ 3 โหมด (3-Mode Smart Auto-Updater)
* **สถานะของต้นทาง:** ❌ **ยังไม่ได้เริ่มต้น (0%)**
* **ฟีเจอร์ใน Koharu-TH:** ระบบจัดการเวอร์ชันและอัปเดตแบบอัตโนมัติ (เลือกโหมดการอัปเดตได้ 3 รูปแบบ: เคว้นการอนุมัติ, ตรวจสอบเงียบเบื้องหลัง, บังคับติดตั้งเมื่อจำเป็น) เพื่อป้องกันผู้ใช้งานใช้งานเวอร์ชันเก่าที่มีข้อบกพร่อง

### 2.5 ระบบความมั่นคงปลอดภัยและการป้องกันแกะโค้ด (Anti-Debugging & Min TLS 1.2 Constraint)
* **สถานะของต้นทาง:** ❌ **ยังไม่ได้เริ่มต้น (0%)**
* **ฟีเจอร์ใน Koharu-TH:** ติดตั้งกลไกความปลอดภัยขั้นสูงตรวจจับการพยายาม Debug หรือแทรกแซงโปรแกรมจากมัลแวร์ภายนอก พร้อมจำกัดระดับโปรโตคอลความปลอดภัยขั้นต่ำไว้ที่ TLS 1.2 สำหรับการเชื่อมต่อคลาวด์แปลภาษาทั้งหมด

### 2.6 ระบบป้องกันข้อมูลรั่วไหลในหน่วยความจำ (Zeroize RAM Memory Protection)
* **สถานะของต้นทาง:** ❌ **ยังไม่ได้เริ่มต้น (0%)**
* **ฟีเจอร์ in Koharu-TH:** ทุกครั้งที่มีการส่งข้อมูล API สำคัญสำหรับการแปล ระบบจะทำลายข้อมูลใน RAM (Zeroize) ทันทีหลังทำงานเสร็จ ป้องกันการดักสแกนข้อมูลส่วนบุคคลหรือคีย์ลับของผู้ใช้งาน

### 2.7 ฐานข้อมูลโปรเจกต์สำรองความร้อน (Thread-Safe Global Standalone Database Fallback)
* **สถานะของต้นทาง:** ❌ **ยังไม่ได้เริ่มต้น (0%)**
* **ฟีเจอร์ใน Koharu-TH:** พัฒนาระบบสำรองฐานข้อมูลส่วนกลางสำหรับโปรเจกต์มังงะเดี่ยว ป้องกันข้อมูลบันทึกแปลสูญหายหากมีการปิดโปรแกรมกะทันหันหรือเกิดการชนกันของเธรดเขียนไฟล์
